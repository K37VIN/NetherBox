// about this project

What is
NetherBox?

A fully automated ML pipeline that takes raw CSV data and produces a production-ready trained model — with zero manual preprocessing, zero model selection, and zero guesswork.

// the problem
ML is too much work

Most data scientists spend 80% of their time on data cleaning, feature engineering, and model selection — not on the actual insight. Every new dataset means the same tedious pipeline written from scratch.

And tools like ChatGPT? They give you answers in a conversation. When the chat ends, everything is gone. No saved model. No reproducibility. No deployment path.

// the solution
One pipeline, any data

NetherBox automates the entire ML workflow. Drop in your CSV, tell it which column to predict, and the pipeline handles everything: cleaning, encoding, scaling, training five models, picking the best one, and packaging it for deployment.

The result is a real, saved model file that lives on your machine — not a chat bubble. Load it tomorrow, next week, or next year. It works.

// tech stack
scikit-learn
ml engine
All preprocessing pipelines and candidate models. ColumnTransformer handles numeric/categorical splits. Cross-validation picks the winner.
FastAPI
rest api
Async Python API with automatic docs at /docs. Handles file uploads, training triggers, prediction calls, and model downloads.
MLflow
experiment tracking
Every training run is logged with CV scores for all 5 models, test metrics, and the model artifact. Browse at localhost:5001.
Docker
containerisation
Multi-stage Dockerfile produces a slim runtime image. Run the entire stack anywhere without environment setup.
// pipeline stages
01
Data Ingestion
data_ingestion.py
Loads your CSV, auto-detects the problem type (classification if target has ≤20 unique values or is string, regression otherwise), and splits into 80/20 train/test with stratification for classifiers.
auto-detect
02
Data Validation
data_validation.py
Checks for missing values, duplicate rows, constant columns, and target nulls. Writes a full JSON report to artifacts/reports/. Warns but doesn't block on minor issues.
report
03
Data Transformation
data_transformation.py
Numeric columns: median imputation + StandardScaler. Categorical columns: most-frequent imputation + OneHotEncoder. High-cardinality columns capped at 15 categories.
sklearn pipeline
04
Model Training
model_trainer.py
Runs 5-fold cross-validation on 5 candidate models in parallel. Scores by F1-weighted for classification, R² for regression. Refits the winner on full training data.
5-fold CV
05
Model Evaluation
model_evaluation.py
Evaluates the best model on held-out test data. Logs all metrics to MLflow. Saves a metrics.json report. Classification: accuracy + F1. Regression: R², MAE, RMSE.
mlflow
06
Model Pusher
model_pusher.py
Bundles the preprocessor and best model into a single AutoMLEstimator pickle. This is the file you download — load it anywhere and call .predict() directly.
export
